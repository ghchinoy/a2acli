# Retrieving Task Status and Artifacts

The `a2acli get` command fetches the current state of a task from an A2A service. It allows you to download any artifacts the agent produced.

## Important Requirements for Agents

When using this command as an automated agent, **you must ALWAYS include the `--no-tui` (or `-n`) flag** to disable the terminal interface and receive JSON output.

## Basic Usage

To retrieve the status of a specific task by its Task ID:

```bash
a2acli get <TaskID> --service-url <URL> --no-tui
```

If the agent requires authentication, provide a token:

```bash
a2acli get <TaskID> --service-url <URL> --token "<TOKEN>" --no-tui
```

### Downloading Artifacts

To automatically save artifacts produced by the task to the filesystem:

```bash
a2acli get <TaskID> --out-dir ./output/ --service-url <URL> --no-tui
```

You can also specify a specific filename to save the artifact to:

```bash
a2acli get <TaskID> --file output.txt --service-url <URL> --no-tui
```

If a task produces multiple artifacts and you use `--file`, the CLI will append an index to the filename for subsequent artifacts (e.g., `output.txt`, `output_1.txt`, `output_2.txt`).

## Parsing the Output

The output will be a JSON object representing the `Task`. Key fields to check:

- `id`: The Task ID.
- `status.state`: The current state (e.g., "TASK_STATE_ACTIVE", "TASK_STATE_COMPLETED", "TASK_STATE_FAILED", "TASK_STATE_REJECTED").
- `artifacts`: An array of artifacts produced by the task. Each artifact has a `name`, `description`, and `parts` containing the actual data or text.

Example output snippet:

```json
{
  "id": "task-12345",
  "status": {
    "state": "TASK_STATE_COMPLETED"
  },
  "artifacts": [
    {
      "name": "report.pdf",
      "description": "The final report",
      "parts": [
        ...
      ]
    }
  ]
}
```

## When to Use This Skill

- When a task is running asynchronously and you need to poll for its completion status.
- To retrieve the artifacts generated by a task that you initiated previously.
- To inspect why a task failed (by checking its status and any error messages in the parts).
